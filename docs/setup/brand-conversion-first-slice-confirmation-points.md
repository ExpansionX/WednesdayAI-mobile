# Brand Conversion First Slice Confirmation Points

## Scope allowed in this slice

This first slice may update paired README product framing, app-visible display and permission copy, and the minimal WednesdayAI backend descriptor path.

Repository documentation uses `WednesdayAI Mobile`. App config display and OS permission copy use the shorter OS-facing name `WednesdayAI`.

The backend descriptor work may add `wednesdayai` as a backend identity only. It must not change transport identity, connection routes, gateway protocol, release identifiers, package names, relay defaults, or external repositories.

## Confirmation-bound surfaces not chosen

This slice does not choose final iOS bundle ID, Android package ID, share extension bundle ID, app group, Expo scheme, Expo slug, Expo owner, EAS project ID, Apple team, store listing IDs, public npm scope, public CLI name, relay domains, worker names, hosted docs/support/legal URLs, store metadata, or YouMind disposition.

Those values remain confirmation-bound until a later human-confirmed migration explicitly changes them.

## Current values that must remain unchanged

Current confirmation-bound values such as `com.expansionx.clawket`, `group.com.expansionx.clawket`, `clawket`, `p697`, `972e845f-da81-44db-a908-24be4ca80288`, `@p697/clawket`, `clawket pair`, and Clawket relay names must remain unchanged unless a later human-confirmed migration says otherwise.

OpenClaw and Hermes compatibility remain explicit. YouMind remains a compatibility or pending-disposition surface in this slice, not a product decision.

## Stop conditions for implementation tasks

Stop if any implementation task would choose a final native ID, package name, public CLI name, relay/domain value, hosted URL, store metadata value, or YouMind disposition.

Stop if a task would require broad Clawket string replacement, image asset renaming, package metadata changes, release config changes, relay config changes, generated native project edits, or external OpenClaw, Hermes, or WednesdayAI-core repository edits.

Stop if README/docs would need to use a name other than `WednesdayAI Mobile`, or app config display/permission copy would need to use a name other than `WednesdayAI`, without a new human-confirmed naming decision.
